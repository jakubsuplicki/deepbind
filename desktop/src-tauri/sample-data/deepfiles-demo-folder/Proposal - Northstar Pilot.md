# Proposal - Northstar Pilot

## Client

Northstar Fabrication is evaluating a local knowledge assistant for its
operations, quality, and field-service teams. The pilot covers one shared
project folder and one month of support.

## Problem

The team stores decisions across meeting notes, supplier emails, spreadsheets,
and saved web pages. Managers want faster answers without sending client or
supplier material to a cloud AI service.

## Proposed Pilot

- Import a controlled folder of project documents into DeepBind.
- Keep the source documents local to the workstation.
- Create local memory notes with safe source-relative provenance.
- Ask scoped questions about the imported folder after approval.

## Success Criteria

| Area | Target |
| --- | --- |
| Import coverage | At least 90 percent of selected files become readable memory |
| Reviewability | Skipped files are visible with plain-language reasons |
| Response quality | Answers cite notes from the imported folder |
| Reversibility | The pilot import can be removed without touching unrelated notes |

## Open Questions

- Should supplier pricing live in a separate restricted folder?
- Which team member approves future import batches?
- Do archive files need child extraction in the next phase?
